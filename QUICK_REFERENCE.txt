🐛 TROUBLESHOOTING

  Issue: Balance not updating
  → Check: grep "Stats broadcaster task started" backend.log
  → Check: grep "Balance cache refreshed" backend.log
  → Check: Browser DevTools → Network → WS → /ws/stats

  Issue: Balance updates slowly
  → Check: grep "Failed to refresh balance cache" backend.log
  → Check: Verify POLYMARKET_API_KEY is set correctly
  → Check: Verify TRADING_MODE=live in .env

  Issue: Balance doesn't match Polymarket
  → Wait 30 seconds for next refresh
  → Check if manual trades were made outside PolyEdge
  → Run: python test_balance_websocket.py (option 2)

💡 HOW IT WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. User WebSocket detects trade confirmation
  2. Triggers immediate balance refresh (1-2s)
  3. Updates shared cache + BotState in DB
  4. Stats broadcaster sends update to frontend (every 1s)
  5. Frontend displays new balance instantly
